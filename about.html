What is powerdelivery?


An open-source toolkit for automating an IT asset release process using Windows PowerShell that enables higher quality and more frequent releases by following Continuous Delivery principles.

Continuous Delivery is a term coined in the principles of the Agile Manifesto and the name of a book by Jez Humble of Thoughtworks and David Farley:

"Our highest priority is to satisfy the customer through early and continuous delivery of valuable software."

If a team's goal is to release to their customers in a production environment more often, they need a technology that does a little more than a vanilla "automated build" tool. They also need this technology to be easy to use and not require knowledge of object-oriented programming or expensive tools.


What does powerdelivery do?


  • Lets you keep environment settings (server names, URLs, file system paths, database names, etc.) in simple YAML text files so the entire team has one place to go to know where stuff is and to change it.

  • Lets you move the parameters you pass to any PowerShell command into configuration files so they can target different environments.

  • Prevents builds that have not been successfully deployed in your development environment from making it to test (also known as staging or UAT) and further on to production.

  • Includes many included functions that do the heavy lifting to deploy common assets on the Microsoft platform (web sites, databases, cubes, and services are examples).

  • Integrates with Microsoft Team Foundation Server to give you both detailed logging and a comprehensive summary page to see what happened when a build was deployed.
